Browser Object Model

 

Every browser window, tab popup, frame, and iframe have a window object.

 

  1. Going Global
    1. In general refer to global variables without using the window object.
    2. Some functions are already global.
  1. Dialogs
    1. The following are methods of the window object: alert(), confirm(), prompt().
    2. These need to be used with care, because they stop the program from running until a response is given. Also users can have settings the prevent them from being used repeatedly.
  1. Browser information
    1. The window object has a navigator property that can find out the browser being used; however, users can change this information making it not a good thing to use.
  1. Location
    1. Using the window.location property you can get information about the URL. For example when used with href.

window.location.href

<< "https://www.sitepoint.com/premium/books/javascript-novice-to-ninja"

 

window.location.href = 'https://www.sitepoint.com/javascript/'

<< "https://www.sitepoint.com/javascript/"

  1. When used with host it will return  the domain of the current URL.

window.location.host

<< "www.sitepoint.com"

  1. Using hostname will do the same.
  2. Using port will return the port name.
  3. Using pathname will return the path the site is using.
  4. Using search you will get a "?" followed by the query string parameters.
  5. Using hash a "#" and fragment identifier if there is one.
  6. Using origin you get the protocol and domain name.
  7. Using reload() you will force the page to reload.
  8. Using assign() you can load another resource from a URL.
  9. Using the replace() method is almost the same as assign(); however, the current page will not be stored.
  10. Using the tostring() you get a sting that contains the entire URL.
  1. Browser history
    1. Using window.history you can access the information from previous visited sites.
      1. Using length with this you get how many pages were visited prior to the current page.
      2. Using .go() you can go to the page prior to or after with the current page being assigned 0.
  1. Controlling Windows.
    1. Use the window.open() method to the assigned page.

const popup = window.open('https://sitepoint.com','

SitePoint','width=400,height=400,resizable=yes');

  1. The close() method will close a window.
  2. Using the moveTo() method you can assign an X and Y coordinate to move to on the screen.

window.moveTo(0,0); // will move the window to the top-left corner of the screen

  1. Use the resizeTo() method to resize the windows dimensions.

window.resizeTo(600,400);

  1. Screen information
    1. You can use window.screen to find the size of the screen that you page is being displayed on.
      1. Use .height to find the height and .width to find the width.

window.screen.height

<< 1024

 

window.screen.width

<< 1280

  1. To find the height and width excluding any operating system menus use .availHeight and .availWidth

window.screen.availWidth

<< 1280

 

window.screen.availHeight

<< 995

  1. Use .colorDepth to find the color bit depth of the screen

window.screen.colorDepth;

  1. The Document Object
    1. Using document.write() simply writes something on the page. If the page has loaded it will completely replace the document.
    2. Using the document.write() is heavily frowned on.
  1. Cookies
    1. Are text files with name/value pairs separated by a ;

"name=Superman; hero=true; city=Metropolis"

  1. Use document.cookie either assign or read the current page cookies.

document.cookie = 'name=Batman'

<< "name=Batman"

document.cookie = 'city=Gotham'

<< "city=Gotham"

 

Or

 

document.cookie:

<< "name=Batman; hero=true; city=Gotham"

  1. You can use split to break a cookie into an array and use a for of loop to get the values.

const cookies = document.cookie.split("; ");

for (crumb of cookies){

    const [key,value] = crumb.split("=");

    console.log(`The value of ${key} is ${value}`);

}

<< The value of name is Batman

The value of hero is true

The value of city is Gotham

  1. Cookie Expiry Dates
    1. Cookies delete once the session is closed by default.
    2. You can set the date for cookies to expire or make them persistent.

const expiryDate = new Date();

const tomorrow = expiryDate.getTime() + 1000 * 60 * 60 * 24;

expiryDate.setTime(tomorrow);

 

document.cookie = `name=Batman; expires=${ expiryDate.toUTCString()}`;

 

  1. Path and Domain of Cookies

document.cookie = 'name=Batman; path=/'

 

Or

 

document.cookie = 'name=Batman; domain=sitepoint.com';

  1. Secure Cookies
    1. ; secure will ensure the cookies are only sent over HTTPS.

document.cookie = 'name=Batman; secure';

  1. Deleting Cookies
    1. Set them to expire in the past.

document.cookie = 'name=Batman; expires=Thu, 01 Jan 1970 00:00:01 GMT';

  1. Shims and Polyfills

The terms shim and polyfill are often used interchangeably. The main difference between them is that a shim is a piece of code that adds some missing functionality to a browser, although the implementation method may differ slightly from the standard API. A polyfill is a shim that achieves the same functionality, while also using the API commands that would be used if the feature was supported natively.
