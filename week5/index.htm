Testing and Debugging

 

  1. Errors, Exceptions, and Warnings
    1. System Error - problem with the system or device
    2. Programmer Error - the program contains incorrect syntax or faulty logic
    3. User Error - User has entered data incorrectly
  1. Exceptions
    1. An error that produces a return value that can then be used to deal with the error
    2. Produces a stack trace. Example index.html:14
  1. Warnings
    1. Occur if there's an error in the code that isn't enough to cause the program to crash
  1. Testing and Debugging importance
    1. Strict Mode - introduced with ECMAScript 5, produces more exceptions and warnings and prohibits the use of some features.
      1. Requires 'use strict' added to the JavaScript file
      2. Can be used with functions examples (function strictly(){'use strict'; or (function() {'use strict';}());
    1. Linting Tools
      1. JS Lint
      2. JS Hint
      3. ES Lint
      4. Helps when programming as team to enforce style guides.
    1. Feature Detection
      1. Use if statements to ensure that a browser will support the feature you want to use. Example: if (window.holoDeck) { virtualReality,activate();) - this would check to see if holoDeck was supported and if it is no supported the program would skip using it and not fail.
      2. Modernizr - library that can be used to detect features of a browser.
    1. Debugger in the Browser
      1. The Trusty Alert
        1. Programmed into the application to look for input that meets a required parameter. If not met then  an alert() will be opened and will require the user to click OK.
      1. Using the console
        1. Console.log or console.trace() will not stop and give an option like alret(); however, you will be able to see what the value is or where the error is by looking at the console in the browser.
  1. Debugging tools
    1. See the documentation of the browser to see it's debugging tools.
    2. Use the debugger keyword to create a breakpoint on the code to help with debugging. Be sure to remove before publishing.
  1. Error Objects
    1. - const error = new Error('something went wrong');
    2. EvalError - only used for backward compatibility
    3. RangeError - when a number is outside an acceptable range
    4. ReferenceError - when a reference does not exist
    5. SyntaxError - when there is an error in the code syntax
    6. TypeError - when the type of value used id incorrect
    7. URIError - when there is a problem encoding or decoding the URI
    8. InternalError - non-standard error when an error occurs in the JavaScript Engine.
  1. Throwing Exceptions
    1. Use the throw statement. - throw new Error('something has gone wrong');
